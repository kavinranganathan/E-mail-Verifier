What we check

Each signal is returned alongside the verdict, so you can set your own threshold.

Syntax

RFC-5322 structure. Catches typos and malformed addresses instantly.

Mail server

Looks up MX records. A domain with no mail server can't receive anything.

Disposable

Flags throwaway providers like Mailinator and 10-minute mail.

Role address

Detects info@, sales@, admin@ — deliverable, but rarely a real person.

Free provider

Notes Gmail, Outlook, iCloud and friends. Informational, never penalized.

Mailbox v2

SMTP mailbox confirmation. Coming next — until then a clean address scores up to 80, capped until we confirm the inbox.

Built for developers

One endpoint, a typed client, and a drop-in widget that verifies as your users type.

# Verify a single address curl -X POST https://your-app.vercel.app/api/verify \ -H 'content-type: application/json' \ -H 'x-api-key: pk_live_...' \ -d '{"email":"alice@example.com"}' # → { "verdict": "risky", "score": 70, "signals": {…}, "reason": "…" }
